# Election Audit & Fraud Detection Tool - System Review & Tutorial

## 1. ภาพรวมระบบ (System Overview)

### 1.1 วัตถุประสงค์
ระบบนี้เป็น **Dashboard สำหรับการตรวจสอบและตรวจสอบความผิดปกติ** ของผลการเลือกตั้ง ส.ส. เขต 2569 โดยเน้นการวิเคราะห์ "ยอดเขย่ง" (ballot discrepancy) ซึ่งเป็นความแตกต่างระหว่างจำนวนผู้มาใช้สิทธิ์เลือก ส.ส. เขต และ ส.ส. บัญชีรายชื่อ ในหน่วยเลือกตั้งเดียวกัน

### 1.2 เทคโนโลยีที่ใช้
| Component | Technology |
|-----------|------------|
| Framework | React 19 |
| Build Tool | Vite 5 |
| Styling | Tailwind CSS 3 |
| Charts | Recharts |
| Icons | Lucide React |
| UI Utilities | clsx, tailwind-merge |

### 1.3 โครงสร้างข้อมูล
ระบบใช้ข้อมูล 3 ชุดหลัก:

1. **plot.json** - ข้อมูลหลัก ประกอบด้วย:
   - ผู้มาใช้สิทธิ์ ส.ส. เขต vs บัญชีรายชื่อ
   - ผลต่างยอดเขย่ง
   - พรรคที่ชนะในแต่ละเขต
   - ผลต่างคะแนนผู้ชนะ (Margin of Victory)

2. **constituency.json** - ข้อมูลรายละเอียดเขต:
   - บัตรเสีย
   - บัตรไม่เลือกผู้ใด (โหวตโน)

3. **referendum.json** - ข้อมูลประชามติ:
   - ผู้มาใช้สิทธิ์ประชามติ (สำหรับเปรียบเทียบ)

## 2. บทช่วยสอนการใช้งาน (User Tutorial)

### 2.1 การเริ่มต้นใช้งาน

1. **เปิดแอปพลิเคชัน** - Dashboard จะโหลดข้อมูลอัตโนมัติจากไฟล์ JSON
2. **รอโหลดข้อมูล** - หากข้อมูลมีขนาดใหญ่ อาจใช้เวลาสักครู่
3. **เลือกแท็บที่ต้องการดู** - มี 3 แท็บหลัก:
   - **ภาพรวมยอดเขย่ง (Overview)** - ดูภาพรวมทั้งหมด
   - **ตรวจสอบความผิดปกติ (Audit)** - ดูเขตที่มีความผิดปกติสูง
   - **ประชามติ (Referendum)** - เปรียบเทียบกับข้อมูลประชามติ

### 2.2 แท็บ: ภาพรวมยอดเขย่ง (Overview)

#### KPI Cards (การ์ดสรุปสำคัญ)
| การ์ด | ความหมาย | การใช้งาน |
|-------|----------|-----------|
| เขตเลือกตั้งทั้งหมด | จำนวนเขตเลือกตั้งทั้งหมดในข้อมูล | ตรวจสอบความครบถ้วนของข้อมูล |
| ยอดเขย่งสัมบูรณ์รวม | ผลรวมของผลต่างสัมบูรณ์ทุกเขต | ดูภาพรวมความผิดปกติ |
| เขตที่มียอดเขย่งสูงสุด | แสดงจังหวัดและเขตที่มียอดเขย่งมากที่สุด | ระบุเขตที่ต้องตรวจสอบด่วน |
| พรรคชนะ ส.ส. เขต มากที่สุด | พรรคที่ได้ที่นั่ง ส.ส. เขตมากที่สุด | ดูการกระจายอำนาจ |
| พรรคชนะปาร์ตี้ลิสต์มากที่สุด | พรรคที่ชนะบัญชีรายชื่อมากที่สุด | เปรียบเทียบกับเขต |

#### วิธีอ่านกราฟแต่ละรูปแบบ

**กราฟที่ 1: จำนวนที่นั่ง ส.ส. เขต แยกตามพรรค**
- แสดงแนวตั้ง (Bar Chart แนวตั้ง)
- แต่ละแท่งแทนพรรค สีตามโลโก้พรรค
- ความยาวแท่ง = จำนวนที่นั่งที่ชนะ
- **วิธีใช้**: ดูการกระจายที่นั่ง เทียบดูว่าพรรคไหนได้เขตไหนบ้าง

**กราฟที่ 2: 15 เขตที่มียอดเขย่งสูงสุด**
- แสดงแนวนอน (Bar Chart แนวนอน)
- สีแดง = บัตรเขต > บัญชีรายชื่อ (บัตรเขตมากกว่า)
- สีน้ำเงิน = บัญชีรายชื่อ > บัตรเขต (บัญชีมากกว่า)
- **วิธีใช้**: ดูเขตที่มีความผิดปกติสูงสุด ระบุเขตที่ต้องสอบสวน

**กราฟที่ 3: ความสัมพันธ์ระหว่างผู้มาใช้สิทธิ์ (Scatter Plot)**
- แกน X = ผู้ใช้สิทธิ์ ส.ส. เขต
- แกน Y = ผู้ใช้สิทธิ์ บัญชีรายชื่อ
- จุดที่อยู่บนเส้นทแยงมุม = ยอดเขย่งน้อย (ปกติ)
- จุดที่ห่างจากเส้น = ยอดเขย่งมาก (ผิดปกติ)
- ขนาดจุด = ระดับยอดเขย่ง (ใหญ่ = เยอะ)
- สีจุด:
  - แดง = ยอดเขย่ง > 1,000
  - ส้ม = ยอดเขย่ง 100-1,000
  - น้ำเงิน = ยอดเขย่ง < 100
- **วิธีใช้**: ดู pattern ความผิดปกติ หากจุดกระจายไม่เป็นเส้นตรงแสดงว่ามีปัญหา

**ตารางข้อมูลรายเขต**
- เรียงตามยอดเขย่งจากมากไปน้อย
- Badge สี:
  - แดง = ยอดเขย่ง > 1,000 (วิกฤติ)
  - ส้ม = ยอดเขย่ง 100-1,000 (เฝ้าระวัง)
  - เขียว = ยอดเขย่ง = 0 (ปกติ)
  - เทา = ยอดเขย่ง < 100 (เล็กน้อย)
- **วิธีใช้**: ค้นหาเขตที่ต้องตรวจสอบ ดูรายละเอียดแต่ละเขต

### 2.3 แท็บ: ตรวจสอบความผิดปกติ (Audit)

**Critical Alerts (การแจ้งเตือนวิกฤติ)**
- แสดงเฉพาะเขตที่มี **ยอดเขย่ง > ผลต่างคะแนนผู้ชนะ**
- หมายความว่า: หากยอดเขย่งนี้เป็นการโกง อาจเปลี่ยนผลผู้ชนะได้
- แสดงจำนวน badge สีแดงบนแท็บ
- **วิธีใช้**: ตรวจสอบเขตที่เสี่ยงต่อการพลิกผล ต้องตรวจสอบด่วน

**กราฟ: ความสัมพันธ์ระหว่างยอดเขย่งและบัตรเสีย (Composed Chart)**
- แท่งแดง (Bar) = ยอดเขย่ง (แกนซ้าย)
- เส้นเทา (Line) = % บัตรเสีย + โหวตโน (แกนขวา)
- **การตีความ**:
  - หากแท่งและเส้นขึ้นพร้อมกัน = อาจมีการทำบัตรเสียมากผิดปกติ
  - หากแท่งสูงแต่เส้นต่ำ = บัตรหายไปโดยไม่มีบัตรเสียรองรับ (สงสัย)
- **วิธีใช้**: หาความสัมพันธ์ระหว่างยอดเขย่งกับบัตรเสีย เพื่อหาหลักฐานการโกง

### 2.4 แท็บ: ประชามติ (Referendum)

**ตารางเปรียบเทียบผู้มาใช้สิทธิ์**
- แสดง 20 เขตที่มีส่วนต่างสูงสุด
- เลือกตั้งเขต vs ประชามติ (ควรใกล้เคียงกัน)
- ส่วนต่าง = บัตรที่มีในเลือกตั้งแต่ไม่มีในประชามติ
- **การตีความ**:
  - ส่วนต่างสูง = มี "บัตรผี" ที่มาใช้สิทธิ์เลือกตั้งแต่ไม่มาประชามติ
  - % ความต่าง > 5% = สงสัย
  - % ความต่าง > 10% = วิกฤติ
- **วิธีใช้**: หาเขตที่มีการใช้สิทธิ์ผิดปกติเทียบกับประชามติ

## 3. การตีความข้อมูล (Data Interpretation Guide)

### 3.1 ยอดเขย่ง (Ballot Discrepancy) คืออะไร?

**คำนิยาม**:
```
ยอดเขย่ง = |ผู้ใช้สิทธิ์ ส.ส. เขต - ผู้ใช้สิทธิ์ บัญชีรายชื่อ|
```

**ทำไมควรเท่ากัน?**
- ในหน่วยเลือกตั้งเดียวกัน ผู้มีสิทธิ์เลือกตั้งคือคนเดียวกัน
- ทุกคนควรได้รับบัตร 2 ใบ พร้อมกัน
- จึงควรมีจำนวนผู้มาใช้สิทธิ์เท่ากัน

**ทำไมถึงต่างกันได้?**
| สาเหตุ | ลักษณะ | ความรุนแรง |
|--------|--------|-----------|
| บัตรชำรุด/หาย | ยอดเขย่งน้อย (<100) | ปกติ |
| ผู้ใช้สิทธิ์ไม่เลือกบัญชีรายชื่อ | เขต > บัญชี | ต่ำ |
| การทุจริต | เขตมากกว่าบัญชีมาก | สูง |
| ซื้อสิทธิ์ขายเสียง | บัญชีมากกว่าเขต | สูง |

### 3.2 Critical District (เขตวิกฤติ) คืออะไร?

**คำนิยาม**: เขตที่มี **ยอดเขย่ง > Margin of Victory**

**ความหมายเชิงการเมือง**:
- หากยอดเขย่งนี้เป็นการโกง อาจเปลี่ยนผู้ชนะได้
- ตัวอย่าง: ที่ 1 ชนะที่ 2 ด้วยคะแนน 500 แต่ยอดเขย่ง 1,000
- หาก 1,000 นี้เป็นการโกง อาจทำให้ที่ 2 ชนะแทน

### 3.3 บัตรเสียและบัตรไม่เลือกผู้ใด (Invalid Ballots)

**สูตรคำนวณ**:
```
% บัตรเสียรวม = (บัตรเสีย + โหวตโน) / ผู้มาใช้สิทธิ์ × 100
```

**ค่าปกติ**: 1-3%
**สงสัย**: > 5%
**วิกฤติ**: > 10%

**ความสัมพันธ์กับยอดเขย่ง**:
- หากบัตรเสียสูงพร้อมยอดเขย่งสูง = อาจมีการทำลายบัตร
- หากยอดเขย่งสูงแต่บัตรเสียต่ำ = บัตรหายโดยไม่มีหลักฐาน

## 4. คู่มือการอ่านกราฟแต่ละประเภท (Chart Types Explained)

### 4.1 Bar Chart (แผนภูมิแท่ง)

**รูปแบบ**: แท่งตั้ง/แท่งนอน
**ใช้สำหรับ**: เปรียบเทียบค่าหลายรายการ
**ตัวอย่างในระบบ**:
- จำนวนที่นั่งแต่ละพรรค
- ยอดเขย่ง 15 อันดับแรก

**วิธีอ่าน**:
1. ดูแท่งที่ยาวที่สุด = ค่าสูงสุด
2. ดูแท่งที่สั้นที่สุด = ค่าต่ำสุด
3. เปรียบเทียบความยาวระหว่างแท่ง
4. ดูสี (ถ้ามี) = กลุ่ม/ประเภท

### 4.2 Scatter Plot (แผนภูมิกระจาย)

**รูปแบบ**: จุดบนระนาบ XY
**ใช้สำหรับ**: ดูความสัมพันธ์ระหว่าง 2 ตัวแปร
**ตัวอย่างในระบบ**: 
- ผู้ใช้สิทธิ์เขต vs บัญชีรายชื่อ

**วิธีอ่าน**:
1. จุดบนเส้นทแยงมุม = ความสัมพันธ์เชิงเส้นตรง (ปกติ)
2. จุดห่างจากเส้น = ความผิดปกติ
3. ขนาดจุด = ค่าที่ 3 (ยอดเขย่ง)
4. สีจุด = ระดับความรุนแรง

### 4.3 Composed Chart (แผนภูมิผสม)

**รูปแบบ**: Bar + Line บนแกน Y คู่
**ใช้สำหรับ**: เปรียบเทียบ 2 ตัวแปรที่มีหน่วยต่างกัน
**ตัวอย่างในระบบ**:
- ยอดเขย่ง (Bar, แกนซ้าย) + % บัตรเสีย (Line, แกนขวา)

**วิธีอ่าน**:
1. Bar สูง = ยอดเขย่งมาก
2. Line สูง = % บัตรเสียมาก
3. ดู pattern: Bar และ Line ขึ้นพร้อมกันหรือไม่

## 5. ตัวอย่างกรณีศึกษา (Case Studies)

### กรณีที่ 1: เขตที่มี Critical Alert
**สถานการณ์**: เขต A มียอดเขย่ง 2,500 ใบ แต่ Margin of Victory เพียง 800 คะแนน
**การตีความ**: 
- หาก 2,500 ใบนี้เป็นการโกง ผู้ชนะอาจเปลี่ยนได้
- ต้องตรวจสอบบัตรทั้งหมดในเขตนี้
- ดูว่ามีบัตรเสียมากผิดปกติหรือไม่

### กรณีที่ 2: ยอดเขย่งสูงแต่บัตรเสียต่ำ
**สถานการณ์**: เขต B ยอดเขย่ง 5,000 แต่ % บัตรเสียเพียง 0.5%
**การตีความ**:
- บัตร 5,000 ใบหายไปโดยไม่มีบัตรเสียรองรับ
- สงสัยว่ามีการยักยอกบัตร
- ควรตรวจสอบกล้องวงจรปิดและบัญชีบัตร

### กรณีที่ 3: ส่วนต่างประชามติสูง
**สถานการณ์**: เขต C มีผู้ใช้สิทธิ์เลือกตั้ง 80,000 แต่ประชามติ 70,000 (ต่าง 10,000)
**การตีความ**:
- มี "บัตรผี" 10,000 ใบในเลือกตั้ง
- หรือมีการปลอมแปลงเอกสาร
- ต้องตรวจสอบรายชื่อผู้ใช้สิทธิ์

## 6. คำแนะนำเพิ่มเติม (Best Practices)

### สำหรับนักวิเคราะห์
1. **เริ่มจาก Critical Alerts** - ตรวจสอบเขตวิกฤติก่อน
2. **ใช้ Scatter Plot** - ดู pattern ความผิดปกติทั้งหมด
3. **เปรียบเทียบหลายแหล่ง** - ใช้ข้อมูลประชามติยืนยัน
4. **ตรวจสอบบัตรเสีย** - ดูกราฟ Invalid Ballots Correlation
5. **ทำรายงาน** - บันทึกเขตที่พบความผิดปกติพร้อมหลักฐาน

### สำหรับผู้สังเกตการณ์
1. **โฟกัสที่ยอดเขย่ง > 1,000** - เขตสีแดงในตาราง
2. **ดูสีในกราฟ** - สีแดงคือความผิดปกติสูง
3. **เปรียบเทียบจังหวัด** - ดูว่ามี pattern เฉพาะพื้นที่หรือไม่
4. **ตรวจสอบพรรค** - ดูว่าความผิดปกติกระจุกตัวที่พรรคใด

### ข้อควรระวัง
- ข้อมูลนี้เป็น **94% ไม่เป็นทางการ** จาก กกต.
- ต้องรอข้อมูลอย่างเป็นทางการเพื่อยืนยัน
- ยอดเขย่งไม่ได้แปลว่ามีการโกงเสมอไป อาจมีสาเหตุอื่น
- ควรใช้ร่วมกับหลักฐานอื่น (กล้องวงจรปิด, พยาน, เอกสาร)
